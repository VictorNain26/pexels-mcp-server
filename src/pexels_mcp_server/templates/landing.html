Pexels MCP server PUBLIC

Free Pexels photo and video search as a Model Context Protocol server. 8 tools + 3 resources + 2 prompts, OAuth 2.1, bring-your-own Pexels API key.

Connect from claude.ai web

  1. Settings → Connectors → Add custom connector.
  2. URL: loading...
  3. Click Connect. A browser tab opens on this server's /setup page asking for your Pexels API key.
  4. Paste your free Pexels API key (get one at pexels.com/api, takes 30 seconds, no credit card) and click Connect. The tab closes automatically.
  5. Activate the connector in a conversation, try: « Find me three landscape photos of Paris on Pexels ».

Your Pexels quota stays yours

Your Pexels API key is bound to your OAuth access token and forwarded to api.pexels.com on every tool call. It lives only in this server's memory, never on disk, and is dropped on token expiry (30 days) or on the next server restart. Every request counts against your Pexels quota (200/hour on the free tier), not anyone else's.

What this server exposes

The three MCP primitives, all read-only and aligned with the MCP spec 2025-11-25.

  • 8 tools (model-controlled): pexels_search_photos, pexels_get_photo, pexels_search_videos, pexels_get_video, pexels_get_collection_media, pexels_get_curated_photos, pexels_get_popular_videos, pexels_get_featured_collections.
  • 3 resources (URI templates): pexels://photo/{id}, pexels://video/{id}, pexels://collection/{id}.
  • 2 prompts: find_hero_image, find_broll — surfaced in the claude.ai connector menu.

Every result includes photographer / uploader_name credit so the agent renders the user-facing answer with the mandatory attribution.

Other MCP clients

Same URL works with Claude Desktop (Settings → Connectors → Add), Claude Code (claude mcp add with --transport http), and the official MCP Inspector. OAuth discovery is automatic via RFC 9728 protected-resource metadata. Power-user setups can skip /setup by sending the Pexels key as an X-Pexels-Api-Key header on every call instead.
